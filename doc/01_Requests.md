# 01. 要求一覧

## 概要

AWS資格試験（DAE等）の学習を支援する個人用Webアプリケーション。
問題演習を通じて知識の定着を図る。

## 要求

### 出題・学習

#### REQ-001: 問題出題機能

問題を出題し、ユーザーが回答できること。

#### REQ-002: 出題モード

以下のパターンで出題できること。すべてのモードにおいて、YAML上の登録順ではなくランダムにシャッフルして出題する。

- **全問ランダム** — 全問題からランダムに出題する
- **カテゴリ別** — 特定のカテゴリに絞って出題する
- **誤答のみ** — 過去に間違えた問題だけを出題する
- **お気に入りのみ** — お気に入りに登録した問題だけを出題する

#### REQ-003: 解説表示

回答後、各選択肢について以下を表示できること。

- 正解の選択肢がなぜ正解なのか（正解理由）
- 不正解の選択肢がなぜ不正解なのか（不正解理由）

#### REQ-004: お気に入り機能

問題にお気に入り（ブックマーク）の印を付けられること。お気に入りの状態はユーザーごとにDBに保存する。

#### REQ-005: 選択肢シャッフル

問題の出題順だけでなく、選択肢の表示順もランダムにシャッフルする。位置で正解を記憶してしまうことを防止する。

#### REQ-006: 出題数の指定

1セッションの出題数を指定できること（例: 10問、20問）。指定しない場合は対象の全問を出題する。

#### REQ-007: 学習進捗ダッシュボード

以下の情報を一覧表示する画面を設ける。

- カテゴリ別の正答率
- 総回答数
- 未着手問題数

### データ

#### REQ-008: 対象資格・複数資格対応

AWS認定資格の学習に対応すること。初期対象はDAE（Data Analytics Engineer）。
今後、他の資格試験も追加していく前提で、資格ごとに問題セットを分離できる構成とする。資格の追加がYAMLファイルの追加と最小限の設定変更で完結するよう、拡張性を考慮する。

#### REQ-009: 問題データ形式

問題データはYAML形式のファイルで管理する。1問あたり以下の要素を1セットとして保持する。

- 問題文
- 選択肢（複数）
- 正解
- 正解理由
- 不正解理由（各選択肢ごと）
- 問題区分（カテゴリ）

#### REQ-010: 回答履歴

ユーザーの回答履歴（どの問題に何を回答したか、正誤）をSupabaseのDBに保存する。出題モード（誤答のみ等）のフィルタリングに使用する。

### インフラ・技術

#### REQ-011: ホスティング構成

- **フロントエンド** — GitHub Pagesで静的サイトとして配信する（SPA）
- **問題データ** — YAMLファイルとしてGitHubリポジトリに格納し、GitHub Pages経由で配信する（DBには格納しない）
- **回答履歴・お気に入り** — SupabaseのDBに保存する

#### REQ-012: 認証・ユーザー管理

SupabaseのGitHub OAuthを利用してユーザー認証を行う（個人利用）。ログインユーザーごとに回答履歴・お気に入りを管理する。ユーザー識別はSupabaseが付与するUIDで行う。厳密な権限管理は不要。

#### REQ-013: 技術スタック

フロントエンドは素のHTML + JavaScriptで構築する（フレームワーク・ビルドツール不使用）。
YAMLのパースにはjs-yamlなどのライブラリをCDNから読み込んで利用する。

#### REQ-014: アニメーション

画面遷移や回答時のフィードバックに軽微なアニメーションを取り入れる。ただし動作の軽快さを最優先とし、素早く短い演出に留める（もっさりした挙動は不可）。

#### REQ-015: レスポンシブデザイン

PC（自宅）とスマートフォン（外出先）の両方で快適に利用できるよう、レスポンシブデザインで構築する。
